## 导出功能 - 前端原始需求说明 (Mock 版)

### 1. 导出方案配置（Mock 存储）

* **内存方案管理**：前端需维护一个 `exportProfiles` 数组对象。
  * **需求**：支持新增、编辑、删除方案。由于是 Mock 版，数据暂存在 `localStorage` 或 `Pinia/Redux` 中。
* **默认预设**：初始化时自动 Mock 两个固定方案：`"系统标准模板"` 和 `"第三方系统对接方案"`。

### 2. 无限层级分类选择（层级逻辑）

针对你的 **无限层级分类树**，前端需具备以下交互能力：

* **递归勾选 (Recursive Selection)**：
  * 勾选父节点时，UI 需提供选项：“仅当前层级” 或 “包含所有子孙节点（无限级递归）”。
  * **Mock 要求**：前端需写一个递归函数，将选中节点及其下所有深度的子节点 ID 拉平（Flatten）为一个待导出列表。
* **层级路径渲染**：在导出预览中，支持将无限层级的路径拼接显示。
  * 示例：`电子料 > 电阻 > 贴片电阻 > ... > 当前分类`。支持自定义分隔符配置。

### 3. 字段映射设计器 (Drag & Drop)

* **动态字段提取**：根据选中的分类节点，Mock 引擎需自动提取出这些分类关联的所有属性并集。
  * **逻辑**：若选中了多个不同层级的分类，需展示它们所有属性的合集（去重后）。
* **表头映射**：
  * 提供双列布局，左侧为系统原字段名（只读），右侧为导出表头 Input（可编辑）。
* **字段排序**：支持拖拽（DnD）改变字段顺序，直接影响 Mock 导出的 JSON 字段顺序。

### 4. 导出转换规则（Mock 计算）

* **前端实时转换计算**：
  * **编码转换**：在导出执行时，前端遍历数据，根据用户配置的规则（前缀/后缀/拼接）动态修改 JSON 中的 `code` 字段。
  * **枚举映射映射**：前端维护一个 `valueMap` 对象。
    * *Mock 逻辑*：`if (item[field] === '01') return 'RED'`。
* **数据脱敏/格式化**：支持配置日期格式（YYYY-MM-DD）及数值精度。

### 5. 导出预览与文件生成（核心交互）

* **实时预览区域**：
  * 基于 `Ant Design Table` 或类似组件，展示经过转换规则处理后的 **前 10 条 Mock 数据**。
  * **必选校验**：若必填字段在 Mock 数据中为空，预览行需标红。
* **前端文件生成**：
  * **技术要求**：使用 `xlsx` 或 `file-saver` 库。
  * **过程**：前端直接将 Mock 处理后的 JSON 数组转换为 `.xlsx` 或 `.csv` 字节流并触发浏览器下载。

---

### 6. 关键 Mock 数据结构示例 (Internal Data Model)

为了支撑无限层级，你的前端 Mock 状态对象应类似：

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `selectedNodes` | `Array<ID>` | 用户在无限层级树中勾选的节点集合 |
| `includeChildren` | `Boolean` | 是否递归导出所有后代 |
| `columnConfig` | `Array<Object>` | `[{ source: 'id', target: '编号', transform: 'xxx' }]` |
| `pathSeparator` | `String` | 默认 `/`，用于处理无限层级的路径展示 |

---

### 7. 特殊交互：针对“无限层级”的性能优化

* **延迟加载预览**：由于层级无限，勾选“包含子孙”可能涉及数千条 Mock 数据。预览区应限制仅处理前 10 条，避免前端计算导致的 UI 卡顿。
* **树形搜索**：在导出页面的分类树中，必须提供搜索过滤，方便用户在无限层级中定位目标节点。
